## 📋 복붙용 프롬프트 (시작)

```
/Users/go/develop/bang-zombie 디렉토리에서 작업해줘.

나는 맥북 웹캠으로 플레이하는 1인칭 슈팅 게임 "빵! 좀비(Bang Zombie)"를 만들고 싶어.
핵심은 웹캠이 내 손을 **권총 모양**으로 인식하고, "빵!" 제스처(엄지 꺾기 또는 검지 튕김)로
좀비를 쏘는 거. 기획서는 이미 작성되어 있으니 먼저 반드시 읽어.

## 먼저 읽을 것
1. docs/기획서.md  ← 전체 기획. 기술 선택, 제스처 판정 로직, 마일스톤 다 있음.
2. README.md

## 작업 원칙
- 기획서의 **M1 → M2 → M3 → M4** 마일스톤 순서대로 진행. 한 마일스톤 끝날 때마다 나한테 데모/확인 요청.
- 스택: Vite + TypeScript + HTML5 Canvas + MediaPipe Hands (Tasks API, WASM 모델).
- 설치 제로가 목표 — 브라우저에서 바로 돌아야 함. `npm run dev` → localhost 접속 → 카메라 권한 허용 → 플레이 가능해야 함.
- 폴더 구조는 기획서 7번 섹션을 따르되, 합리적이면 조정 가능.
- MediaPipe는 공식 문서 먼저 확인 (Context Hub / document-specialist 활용).
- 과잉 엔지니어링 금지. MVP 먼저, 폴리싱 나중.
- 코드에 주석은 "왜"가 비자명할 때만. 길어지면 삭제.

## M1 목표 (지금 당장 할 것)
1. `npm init` + Vite + TS 세팅, `.gitignore`, `package.json`
2. `index.html` + `src/main.ts` — 웹캠 비디오 엘리먼트 + 캔버스 오버레이
3. MediaPipe Hands 연결 — `@mediapipe/tasks-vision` 사용
4. 21개 랜드마크를 캔버스에 오버레이 그리기 (좌우 반전 = 거울 모드)
5. 기획서 4.2절 기준으로 "권총 포즈" 판정 → 화면에 "GUN POSE: YES/NO" 텍스트 표시
6. `npm run dev` 돌려서 내가 직접 확인할 수 있는 상태로 만들기

## 제약
- 실행은 백그라운드로 돌리고, 나한테 `http://localhost:xxxx` 주소 알려줘.
- 카메라 권한은 내가 직접 허용할 테니 그 부분은 브라우저에 맡기고 에러 핸들링만 잘 해둬.
- M1 끝나면 멈추고, 내가 눈으로 확인한 후에 M2 진행.

자, 시작하자. 먼저 기획서부터 읽어.
```

## 트러블슈팅 팁 (새 세션에 공유해도 됨)
- 맥북 Safari보다 Chrome이 MediaPipe WASM 지원 안정적.
- `getUserMedia`는 **localhost 또는 HTTPS**에서만 동작. Vite dev server는 localhost라 OK.
- MediaPipe 모델 파일(.task)은 CDN에서 받거나 `public/` 에 두고 정적 서빙.
- 맥북 Apple Silicon에서 M1~M4 정도는 30~60fps 충분히 나옴.
