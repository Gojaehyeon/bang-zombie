# 빵! 좀비 (Bang Zombie) — 기획서

## 1. 한 줄 요약
맥북 웹캠으로 내 손을 총으로 인식해서, "빵!" 제스처로 좀비를 쏘는 브라우저 기반 1인칭 슈팅 게임.

## 2. 왜 만드나 (Why)
- 키보드/마우스 없이 **몸으로 노는** 물리 게임 경험
- 맥북 웹캠 + MediaPipe만으로 설치 없이 바로 플레이 (브라우저 하나면 끝)
- 짧게 치고 빠지는 아케이드성 — 1판 3~5분, 리플레이성 높음

## 3. 핵심 경험 (Core Loop)
1. 웹캠 앞에서 손을 **권총 모양**으로 만든다 (검지 뻗고, 엄지 위로, 나머지 접음)
2. 좀비가 화면 좌/우/위에서 스폰되어 조준점(= 검지 끝) 쪽으로 다가옴
3. 검지 끝을 좀비에 조준한 뒤 **"빵!" 제스처** — 엄지를 아래로 탁 꺾는 / 검지를 살짝 위로 튕기는 동작 — 으로 발사
4. 좀비 처치 → 점수 획득 → 웨이브 진행 → 체력 0이 되면 게임 오버

## 4. 제스처 인식 설계 (핵심 기술)
### 4.1 라이브러리
- **MediaPipe Hands** (Google) — 웹캠 영상에서 21개 손 랜드마크를 실시간 추출. 브라우저에서 WASM으로 돌아감. 맥북 M1 이상에서 30~60fps 나옴.
- 대안: TensorFlow.js HandPose (백업)

### 4.2 "권총 포즈(Gun Pose)" 판정
다음 조건을 **동시에** 만족하면 권총 포즈로 인정:
- 검지(index): 펴짐 — landmark 8(tip)이 landmark 6(pip)보다 손목에서 더 멀리
- 중지/약지/새끼: 접힘 — 각 tip이 해당 mcp보다 손목에 더 가까움
- 엄지: 위로 뻗음 — landmark 4(tip)가 landmark 2(mcp)보다 위쪽(Y 작음)

### 4.3 "빵!" 발사 제스처 판정 (두 옵션, 프로토타입에서 둘 다 구현 후 선택)

**옵션 A — 엄지 꺾기 (Thumb Snap, 권장)**
- 권총 포즈 유지 중, 엄지 끝(4)이 **아래로 내려가는 속도**가 임계치 초과
- 즉, thumb tip의 Y값이 최근 3프레임 기준 빠르게 증가 (`dy/dt > threshold`)
- 실총 격발과 가장 직관적으로 닮음

**옵션 B — 검지 끝 튕김 (Index Flick)**
- 권총 포즈 유지 중, 검지 끝(8)이 **위로 튕기는** 속도가 임계치 초과
- 사용자 요청 원문("손가락 끝이 살짝 올라가는") 그대로 구현

### 4.4 오발/연사 방지
- 발사 후 200ms 쿨다운 (debounce)
- 포즈가 깨진 상태에서 한번 리셋되어야 다음 발사 인정 (자동연사 금지)
- 영상 좌우 반전 (거울 모드) — 사용자가 직관적으로 조준 가능

### 4.5 조준점
- 권총 포즈의 **검지 끝(landmark 8)** 좌표를 게임 좌표계로 변환 → 크로스헤어
- 약간의 지수이동평균(EMA) 스무딩으로 손떨림 보정

## 5. 게임 디자인

### 5.1 화면 구성
- 전체화면 캔버스
- 좌상단: 점수, 웨이브, 체력, 남은 탄약
- 우상단: 웹캠 프리뷰 (작은 PIP, 손 랜드마크 오버레이)
- 중앙: 게임 월드 + 좀비 + 크로스헤어

### 5.2 좀비
- 스프라이트 또는 심플 도트/실루엣 (MVP는 심플)
- 3종:
  - **일반좀비**: HP 1, 느림 (점수 10)
  - **러너**: HP 1, 빠름 (점수 20)
  - **탱커**: HP 3, 느림 (점수 50)
- 화면 가장자리에서 스폰 → 크로스헤어(=플레이어) 방향으로 이동
- 접근 거리 안으로 들어오면 플레이어 HP -1

### 5.3 웨이브 & 난이도
- 웨이브당 좀비 N마리, 웨이브 넘어갈수록 +스폰수, +속도
- 체력 3, 사이 재충전 없음
- 무한모드 + 점수 기록 (localStorage 최고점)

### 5.4 탄약 & 리로드
- 탄창 6발
- 탄창 비면 **주먹 쥐기** 1초 = 리로드 (제스처로 리로드까지)

### 5.5 사운드
- 빵! 발사음 / 빈총 찰칵 / 좀비 신음 / 피격음 / BGM (로열티프리)

## 6. 기술 스택
| 영역 | 선택 | 이유 |
|---|---|---|
| 런타임 | 브라우저 (Chrome/Safari) | 설치 無, 맥북 웹캠 직접 접근 |
| 언어 | TypeScript | 타입 안전 + 최신 ES |
| 렌더링 | HTML5 Canvas 2D | MVP엔 오버스펙 피함, WebGL은 후순위 |
| 손 인식 | MediaPipe Hands (Tasks API, WASM) | 실시간 + 정확도 OK + 무료 |
| 번들러 | Vite | 제로 컨피그, HMR 빠름 |
| 패키지 | pnpm 또는 npm | — |
| 배포 | 정적 파일 (로컬 실행 우선, 나중에 GitHub Pages) | 카메라 권한 위해 HTTPS or localhost 필요 |

## 8. 마일스톤

### M1 — 손 인식 프로토타입 (반나절)
- Vite + TS 세팅
- MediaPipe Hands 연결, 웹캠 영상 + 랜드마크 오버레이 표시
- 권총 포즈 판정 콘솔 로그

### M2 — 발사 제스처 (반나절)
- "빵!" 제스처 A/B 옵션 둘 다 구현
- 발사 이벤트 → 화면에 탄착점 표시
- 쿨다운/디바운스
- 사용자(=메이커)가 A/B 체감 테스트 후 하나 채택

### M3 — 좀비 MVP (하루)
- 좀비 1종 + 스폰 + 이동 + 피격/사망
- 크로스헤어 조준
- 점수, 체력, 게임오버

### M4 — 웨이브 & 폴리싱 (하루)
- 좀비 3종, 웨이브, 탄창/리로드
- 사운드
- 최고점 저장

### M5 — 런칭 (선택)
- GitHub Pages 배포
- 간단한 랜딩(어떻게 손을 들고 있어야 하는지 튜토리얼)

## 9. 리스크 & 완화

## 10. 성공 기준 (MVP)
- [ ] 맥북 웹캠으로 내 손이 권총 포즈로 인식됨
- [ ] "빵!" 제스처로 크로스헤어 위치에 발사 이벤트 발생
- [ ] 좀비 스폰되고, 조준해서 쏘면 죽음
- [ ] 체력 다 까이면 게임오버 + 점수 표시
- [ ] 새 판 시작 가능
- [ ] 메이커가 "재밌다"고 느낌 (정성 기준)
