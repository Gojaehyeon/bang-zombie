## 현재 상태 (2026-04-16)

### 배포

### 완료된 기능
- **손 인식**: MediaPipe HandLandmarker, 3D 거리 기반 권총 포즈 판정 (검지 전방 포즈 포함)
- **얼굴 인식**: MediaPipe FaceLandmarker, 눈 깜빡임(윙크) 발사
- **커서**: 검지 끝 좌표 → 2.6배 증폭 + EMA 스무딩
- **쌍권총 모드**: 500점 돌파 시 자동 해제, 양손 독립 커서, 동시 발사, BGM 전환
- **좀비 3종**: 일반(초록), 러너(빨강), 탱커(보라) — 커스텀 스프라이트
- **무한 스폰**: 시간 기반 점진 난이도 (웨이브 제거됨)
- **난이도**: Easy/Hard (속도 배수 + 스폰 간격 배수)
- **리더보드**: Supabase (프로젝트 `nqbaxfphnxqkhwgaxxed`), 난이도별 상위 100명
- **UI**: 레트로 CRT 스타일 (VT323 폰트, 스캔라인, 초록 인광)
- **메인화면 탭**: PLAY / RANKING / HOW TO
- **손가락 UI 조작**: 메뉴에서 손 커서 + 윙크 클릭
- **음악 4트랙**: 메뉴(`bgm.mp3`), 게임(`bgm_game.mp3`), 쌍권총(`bgm_dual.mp3`), 게임오버(`bgm_3.mp3`)
- **SFX**: 발사(WebAudio 합성), 처치(`item.mp3`), UI 클릭(`click.mp3`)
- **사운드 토글**: 우측 상단 SOUND ON/OFF
- **로딩 프로그레스 바**: 카메라 → 모델 → 스프라이트 순차 로딩 표시

### 기술 스택
- Vite + TypeScript + HTML5 Canvas
- MediaPipe Tasks Vision (HandLandmarker + FaceLandmarker, WASM, CDN)
- Supabase (PostgreSQL, RLS, anon key 사용)
- Electron (데스크톱 앱용)
- Vercel (정적 배포)

## 다음 작업: 모바일 최적화

### 핵심 과제
모바일에서는 **웹캠 손 인식 + 윙크 발사**가 그대로 동작하지만 UX 최적화 필요:

### 1. 터치 입력 대안
- 모바일은 화면을 보면서 손을 카메라에 들기 어려움
- **옵션 A**: 전면 카메라 + 손 인식 유지 (태블릿에선 OK)
- **옵션 B**: 터치 조작 모드 추가 — 손가락 드래그로 조준, 탭으로 발사
- **옵션 C**: 기울기(자이로) 조준 + 탭 발사
- 시작 화면에서 조작 모드 선택하게

### 2. 해상도 / 레이아웃
- `canvas` 크기를 디바이스 비율에 맞게 조정
- HUD 텍스트 크기 반응형 (vmin 기반)
- 메뉴 오버레이 모바일 너비 대응 (현재 560px 고정)
- VT323 폰트 모바일에서도 로딩 확인

### 3. 성능
- MediaPipe 모델 Lite 버전 사용 검토 (현재 float16)
- 모바일에서는 FPS 타겟 낮추기 (30fps)
- 스프라이트 이미지 압축 (현재 1MB+ PNG → WebP 변환)

### 4. 카메라 방향
- 모바일 전면 카메라 `facingMode: "user"` 확인
- 세로/가로 모드 처리 (landscape 강제 권장)

### 5. 오디오
- iOS Safari는 AudioContext를 사용자 터치 후에만 resume 가능
- `unlockAudio()`가 터치 이벤트에서도 호출되는지 확인

### 6. PWA (선택)
- `manifest.json` + 서비스워커 추가하면 홈 화면 추가 가능
- 풀스크린 모드 지원

### 구현 순서 제안
1. 터치 조작 모드 (드래그 조준 + 탭 발사) — 가장 임팩트 큼
2. 반응형 UI 대응
3. 스프라이트 WebP 변환 + 이미지 최적화
4. landscape 강제 + 풀스크린
5. PWA manifest
